# PPT Generate

`ppt-generate` 是一个面向 Codex 的学术 PPT 生成 Skill。它可以根据论文、源 PPT、数据文件和参考风格图，制作页数可变、16:9、中英文混排的学术汇报，并输出可编辑的 `.pptx` 文件。

## 主要特点

- 页数根据汇报时长、内容结构和证据量决定，不固定为 12 页。
- 先生成四套风格预览，每套只展示四个代表页，减少等待时间。
- 默认直接制作混合可编辑 PPT：主要文字、基础图形和图表使用 PowerPoint 原生元素。
- 复杂背景、机制插图和装置合成等区域可使用 Codex 内置图像生成能力。
- 支持完整图片版后再还原的高保真模式。
- 支持逐页任务记录、失败隔离和断点续作。

## 安装

将仓库克隆到 Codex 的 Skills 目录：

```bash
git clone https://github.com/livehc/ppt-generate.git ~/.codex/skills/ppt-generate
```

如果已经安装，可更新到最新版：

```bash
git -C ~/.codex/skills/ppt-generate pull
```

重新打开 Codex 任务后即可使用。

## 最简单的使用方法

上传论文、PPT、PDF、数据文件或参考图片，然后在 Codex 中输入：

```text
使用 $ppt-generate，根据我上传的文献和参考图制作一份组会汇报PPT。
页数请根据内容自动确定，采用16:9、中英文混排。
```

也可以直接使用自然语言触发：

```text
请用三步法PPT流程，把这篇论文整理成一份可编辑的学术汇报PPT。
```

## 标准工作流程

### 1. 内容规划与风格预览

Skill 会先完成整套 PPT 的内容骨架，再选择四个代表页：

1. 封面或总览页
2. 背景、科学问题或比较页
3. 方法、流程或机制页
4. 关键结果或数据图表页

随后生成 A、B、C、D 四套 2×2 风格预览。四套使用相同内容，但在配色、网格、留白、图表语言和视觉母题上有所区别。

看到预览后回复：

```text
选择第 C 套，请继续制作完整PPT。
```

### 2. 生成可编辑 PPT

默认使用快速模式：

- 标题、正文、数字、表格、基础图表、线条、箭头和简单形状尽量保持可编辑。
- 论文原始图表和实验图片优先从来源文件中提取，不重新生成。
- 只有复杂视觉区域才使用图片素材。
- 完成后会渲染全稿并检查页数、比例、溢出、裁切和跨页一致性。

### 3. 可选的高保真模式

如果需要先查看完整图片视觉稿，再还原为可编辑版本，可以明确提出：

```text
使用 $ppt-generate 的高保真模式：先生成完整图片版PPT，确认后再进行混合可编辑还原。
```

该模式会先逐页生成高清视觉稿并合成图片版 PPTX，确认后再采用“复杂视觉保真 + 主要文字可编辑”的策略进行还原。

## 常用提示词示例

### 根据论文制作组会汇报

```text
使用 $ppt-generate，根据我上传的论文制作一份组会汇报。
请自动确定合理页数，采用16:9和中英文混排，重点呈现研究背景、科学问题、方法、关键结果、机制模型、创新点和总结展望。
```

### 参考现有 PPT 的视觉风格

```text
使用 $ppt-generate，以我上传的源PPT作为内容来源，以参考图片作为视觉风格依据。
先生成四套风格预览，每套只展示四个代表页，等待我选择后再制作完整可编辑PPT。
```

### 指定页数或汇报时长

```text
使用 $ppt-generate，把这些资料整理成一份约15分钟的学术汇报，页数由你根据内容决定。
```

或：

```text
使用 $ppt-generate，制作一份18页的项目汇报PPT。
```

### 修改已有 PPT

```text
使用 $ppt-generate，在保留现有内容准确性的前提下，重新设计我上传的PPT。
先给我四套四页风格预览，选择后输出可编辑版本。
```

## 建议提供的材料

- 论文、文献或研究报告
- 原始 PPT、PDF、Word 或 Excel 数据
- 希望模仿的风格截图
- 必须保留的图表、Logo、照片和引用
- 汇报场景、时长、语言和听众背景
- 必须出现或必须避免的内容

材料越完整，PPT 的内容准确性和视觉一致性越高。

## 输出内容

根据所选模式，最终通常包括：

- 可编辑的 PowerPoint 文件
- 全稿缩略浏览图或逐页预览图
- 页面元素编辑性说明
- 图片素材与原生元素清单
- 字体替代、视觉近似数据和其他限制说明

## 注意事项

- Codex 内置图像生成模式不要求用户另行提供本地 OpenAI API Key。
- 图像模型不应用于生成精确数据、公式、DOI、统计符号或长段文字。
- 无法从来源准确反推的数据只按视觉比例近似，并应明确标注。
- 图片版 PPT 主要用于视觉确认；需要后续修改时，应选择混合可编辑版本。
